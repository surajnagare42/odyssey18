General Rules:(CODE ZILLA)


->The decision taken by the co-ordinator/volunteers will be final.
->Use of other electronic devies is prohibited
->The participants should be present on the given time
->The Number of participants is 2

Round-1: C Yourself


->This Round Consists 25 questions and 5 questions for the tie breaker if any Tie's.
->The time limit for this round is 30 Minutes.
->There is no Negative marking for this round.
->Double marking of the answers will not be considered.


Round-2: Blind coding


-> The teams will be given same piece of code.
-> The teams will type the code with MONITOR SWITCHED OFF.
-> Once the teams are finished typing, the coordinators will compile and run the program.
-> Distractions in the form of Loud Music will be played throughout the round.
-> No Switching the monitor on in any situation. Doing so is liable for Disqualification



Round-3: Question Hunt


-> Initially, a jumbled code is given to every team.
-> The team will have to rearrange the code, and execute, to find a password.
-> The team on acquiring password, will be given another jumbled code.
-> This code, on execution will give the location for The Final Problem.
-> To open The Final Problem, the password acquired in the first code is to be used.
-> No lines should be deleted or added to the code, and no line can be commented.
->Doing so will result in Disqualification
-> If no teams generate the output within the given time, the team with less number of errors will be selected for the next round.



Round-4: Confused Coder


-> The mapping of a selected few keyboard keys will be changed to different keys.
-> The remapping of the keys will be displayed throughout the round.
->The teams will be given a problem statement, which is to be solved.
-> The remapped keys will confuse the teams. So the team which executes the program without succumbing to confusion, proceeds to the final round.
-> If the teams donâ€™t generate the output within the given time, the team with less number of errors will be selected.



Round-5: Programmer's Karma


-> The teams will be given 5 problem statements, from which they'll have to choose one.
-> This statement of choice will be given to another team based on random draws.
      The same team will give your team a problem statement of their choice.
-> The teams have to code and get the desired output for the opponent's problem
      statment, AS WELL AS the problem statement they've chosen for the other team.
-> The team among the two teams swapping, who execute both the codes successfully,
       will win.
-> Remember, What you give is what you get. #Karma
